% Option names follow the markdown package v3.x.
\usepackage[
  citations,          % [@key] citation syntax
  hybrid,             % pass raw LaTeX through (e.g. \zcref below)
  notes,              % [^label] referenced footnotes
  inlineNotes,        % ^[inline footnote text]
  pipeTables,         % | a | b | style tables
  tableCaptions,      % : Caption text below a table
  tableAttributes,    % {#id} identifier on a table caption
  hashEnumerators,    % 1) 2) 3) style enumerators
  taskLists,          % - [ ] / - [x] checkbox lists
  headerAttributes,   % {#id} identifier on a heading
  linkAttributes,     % {#id} attributes on links/images
  relativeReferences, % <#id> to reference an id defined via {#id}
]{markdown}

% --- Bibliography: author-year via biblatex/biber ---

% How the labels work (answer to "can't we use the markdown labels?"):
% The markdown package turns `{#id}` (on headings, images and tables) into a
% *standard* \label{id}. zref-clever's `labelhook` (on by default) makes every
% \label additionally a zref label, so \zcref{id} works out of the box with
% markdown's own identifiers -- no raw-LaTeX \zlabel or manual bridge needed.

% --- Clickable everything: load hyperref LAST -----------------------------
% Makes the TOC, \zcref, native <#id> refs, [@citations] and footnotes clickable.
% zref-clever auto-loads zref-hyperref once it sees hyperref, so \zcref links
% need no extra setup. `hidelinks` keeps links clickable without coloured boxes
% (swap for `colorlinks=true` if you prefer visible coloured links on screen).
\usepackage[hidelinks, unicode, pdfusetitle]{hyperref}

# Introduction {#cha:introduction}

This entire book content is written in **Markdown**. Top-level headings (`#`)
automatically become chapters, `##` become sections, matching the `scrbook`
document class. This heading carries the Markdown id `{#cha:introduction}`,
from which the package generates a `\label`.

A citation looks like this [@mwe], and here is a footnote.^[This is an inline
footnote.] Referenced footnotes work as well[^detail].

[^detail]: This is a more detailed footnote at the end of the section.

Task lists (`taskLists`) are possible too:

- [x] an already-completed item

As shown in \zcref{fig:example}, images can be referenced directly from
Markdown. The plain Markdown reference `<#fig:example>` yields only the bare
number (<#fig:example>), without the type name Figure.
